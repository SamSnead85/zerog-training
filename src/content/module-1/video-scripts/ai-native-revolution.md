## Scene 3: The Mindset Shift (2:30 - 4:00)

### Visuals
- Before/after thinking animations
- Developer persona transformations

### Narration
"The AI-Native mindset comes down to three shifts."

### Shift 1: From Typing to Directing
"Stop being the typist. Start being the director. You describe what you want, guide the implementation, and verify the result."

### Shift 2: From Solving to Delegating
"You don't need to solve every problem from scratch. Figure out what to solve, then delegate the how to AI."

### Shift 3: From Perfect Knowledge to Just-in-Time Learning
"You don't need to memorize APIs. You need to know how to ask the right questions and validate answers."

## Scene 5: Practical Example (6:30 - 8:30)

### Visuals
- Real code development sequence
- Split screen: old way vs AI-native way

### Narration
"Let's see this in action. Say you need to build a REST API endpoint that validates user input, stores it in a database, and sends a notification."

### Old Way
```
Time: 2-3 hours
- Research: "FastAPI request validation best practices"
- Write: Schema classes, endpoint, database calls
- Debug: Fix typos, missing imports
- Test: Manual testing, write tests
- Document: Write docstrings and API docs
```

### AI-Native Way
```
Time: 30-45 minutes
1. Describe to AI: "Create FastAPI endpoint for user registration with 
   Pydantic validation, SQLAlchemy storage, and SendGrid notification"
   
2. Review generated code (2 min)

3. Refine: "Add rate limiting and input sanitization"

4. Let AI generate tests

5. Review and deploy
```

### Key Insight
"You're still thinking. You're still reviewing. You're still making decisions. But you're not typing boilerplate."
